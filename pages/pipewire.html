PipeWire Update to 1.0

Latest Updates

PipeWire, designed to replace the frequently problematic PulseAudio, has now reached version 1.0 with substantial enhancements and the customary bug fixes.

Chances are, your Linux distribution utilizes PipeWire, the subsystem responsible for managing audio and video on contemporary Linux desktop systems. After a prolonged wait, version 1.0 has finally been released.

Version 1.0 introduces default jackdbus support, maintains API/ABI compatibility, resolves the jitter issue in ALSA (especially when in IRQ mode), brings numerous bug fixes to various modules, significantly enhances Bluetooth LC3 codec compatibility, JACK transport, and time handling, optimizes buffer re-use with JACK, improves socket permissions, offers MIDI event recording preview in Ardour, enhances ALSA's resume from suspend functionality, and much more.

While PipeWire 1.0 doesn't introduce an extensive array of new features, it includes an option to expose ALSA controls as prop parameters and adds support for XDG-based directories when loading ACP configurations.

Furthermore, the update incorporates pause and resume support in pipe-tunnel, introduces new linear/clamp/recip/exp/log/mult/sing plugins filter-chain, allows handling of NULL values from mmap_areas in the ALSA plugin, and includes uclamp support (enabling the PipeWire scheduler to make more informed decisions regarding task placement).

For detailed information, refer to the PipeWire GitHub page detailing the "El Presidente" release, and await the update to the latest version of the sound subsystem in your preferred Linux distribution.

Source : PipeWire 1.0 Officially Released - Linux Magazine
